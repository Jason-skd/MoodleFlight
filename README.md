# MoodleFlight  "lry"飞行

## 解决痛点
#### 1. 在moodle观看视频，缺乏自动化工具，在播放完成时自动切换下一个视频
#### 2. 缺乏断点续连功能
#### 3. 需要手动复制粘贴网址
#### 4. 脚本难以满足美观、系统、有计划的需求

## 本项目的解决方案
#### 1. 参考现有脚本的先进之处，利用现代化的Playwright框架，结合kotlin的现代化软件开发能力，实现自动化观看视频功能
#### 2. 引入「计划」系统，在行动前要求用户制定计划
* 通览视频总时长，呈现进度条
* 通过json或其他数据保留方式，免去对已完成计划的重复遍历，实现断点续连功能
* 在只需观看定量课程的规则下，自动选择时长最短的视频。
#### 3. 加入日志系统，对未能处理的异常跳转利用ChromeDevTools截屏并记录
#### 4. 虽然本项目是后端程序，暂依赖命令行交互，后续有计划开发ui界面
